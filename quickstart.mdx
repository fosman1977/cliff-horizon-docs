## Prerequisites

- Python 3.10+ with `venv`
- Node.js 18+ and npm
- `uvicorn` will be installed via requirements

---

## 1. Engine setup

```bash
# Clone and enter the repo
git clone https://github.com/fosman1977/cliff-horizon-engine.git
cd cliff-horizon-engine

# Create and activate venv
python3 -m venv venv
source venv/bin/activate   # Windows: venv\Scripts\activate

# Install dependencies
pip install -r requirements.txt

# Start the engine on port 8001
uvicorn engine.app.main:app --reload --port 8001
```

The engine is now running at `http://localhost:8001`. Visit `http://localhost:8001/docs` for the auto-generated OpenAPI UI.

---

## 2. Dashboard setup

The active dashboard is `dashboard-v2/` (not the legacy `dashboard/`).

```bash
cd dashboard-v2
npm install
npm run dev
# → http://localhost:5173
```

---

## 3. First API call

Verify the engine is running by listing configured sites:

```bash
curl http://localhost:8001/api/sites
```

Example response:

```json
[
  { "id": "rize_tr", "name": "Rize, Turkey", "peril": "rain", "lat": 41.02, "lon": 40.52 },
  { "id": "istanbul_tr", "name": "İstanbul, Turkey", "peril": "flood", "lat": 41.01, "lon": 28.97 },
  { "id": "dubai_ae", "name": "Dubai, UAE", "peril": "flood", "lat": 25.20, "lon": 55.27 },
  { "id": "bankot_in", "name": "Bankot Creek, India", "peril": "rain", "lat": 17.94, "lon": 73.27 }
]
```

Get a rain/work-stop quote:

```bash
curl -X POST http://localhost:8001/api/quote \
  -H "Content-Type: application/json" \
  -d '{
    "site": "rize_tr",
    "strikeDays": 10,
    "capDays": 30,
    "burnPerDay": 5000,
    "load": 0.35,
    "threshold": 10
  }'
```

## 4. Remote / SSH dev box

If the engine runs on a remote machine (Studio dev box), start it with `--host` so it binds to all interfaces:

```bash
uvicorn engine.app.main:app --reload --port 8001 --host 0.0.0.0
# Dashboard:
cd dashboard-v2 && npm run dev -- --host
```

Then access via:
- **LAN:** `http://192.168.0.44:5173` / `http://192.168.0.44:8001`
- **Tailscale:** `http://100.90.92.110:5173` / `http://100.90.92.110:8001`

<Note>
`localhost` URLs only work inside the SSH session itself. Always use the LAN or Tailscale address when sharing the demo with someone else.
</Note>

## 5. Verify calibration proof

```bash
curl http://localhost:8001/api/proof/rize_tr
```

Returns a reliability diagram, ECE, and Brier decomposition for the site. See [Proof & Track](/api/proof-track) for how to interpret the output.
